# 需求文档

## 简介

本系统是一个基于 WinForm 和 SQL Server 的学生成绩管理系统，用于学校教务管理人员管理学生信息、教师信息、课程信息和成绩数据。系统提供成绩录入、查询、统计分析等核心功能，帮助管理员高效管理学生成绩和基础数据。

## 术语表

- **系统**: 学生成绩管理系统
- **管理员**: 系统管理员，可以管理所有数据和功能
- **学生**: 在校学生的基本信息和成绩数据
- **教师**: 教师的基本信息
- **课程**: 学校开设的教学科目
- **成绩**: 学生在某门课程中获得的分数
- **班级**: 学生所属的行政班级
- **学期**: 成绩记录的时间范围

## 需求列表

### 需求 1

**用户故事：** 作为管理员，我希望能够管理学生基本信息，以便维护准确的学生档案

#### 验收标准

1. 当管理员选择添加学生功能时，系统应显示学生信息输入表单，包含学号、姓名、性别、出生日期、班级编号和联系方式字段
2. 当管理员提交有效的学生信息时，系统应将学生记录保存到数据库并显示成功确认信息
3. 当管理员按学号或姓名搜索学生时，系统应显示匹配的学生记录及所有详细信息
4. 当管理员选择一条学生记录并选择编辑功能时，系统应允许修改除学号外的学生信息
5. 当管理员选择一条学生记录并选择删除功能时，系统应在允许删除前验证该学生不存在成绩记录

### 需求 2

**用户故事：** 作为管理员，我希望能够管理班级信息，以便组织学生的班级归属

#### 验收标准

1. 当管理员选择添加班级功能时，系统应显示班级信息输入表单，包含班级编号、班级名称和年级字段
2. 当管理员提交有效的班级信息时，系统应将班级记录保存到数据库并显示成功确认信息
3. 当管理员查看班级列表时，系统应显示所有班级及每个班级的学生人数
4. 当管理员选择一个班级并选择编辑功能时，系统应允许修改班级名称和年级
5. 如果班级中有在读学生，则系统应阻止删除该班级记录

### 需求 3

**用户故事：** 作为管理员，我希望能够管理课程信息，以便维护学校的课程目录

#### 验收标准

1. 当管理员选择添加课程功能时，系统应显示课程信息输入表单，包含课程编号、课程名称、学分和教师编号字段
2. 当管理员提交有效的课程信息时，系统应将课程记录保存到数据库并显示成功确认信息
3. 当管理员查看课程列表时，系统应显示所有课程及分配的教师姓名
4. 当管理员选择一门课程并选择编辑功能时，系统应允许修改课程名称、学分和分配的教师
5. 如果课程存在成绩记录，则系统应阻止删除该课程记录

### 需求 4

**用户故事：** 作为管理员，我希望能够录入学生成绩，以便记录学生的学习成果

#### 验收标准

1. 当管理员选择成绩录入功能时，系统应显示所有课程列表供选择
2. 当管理员选择一门课程和学期时，系统应显示所有学生列表及已有成绩（如果有）
3. 当管理员输入或修改 0 到 100 之间的成绩值时，系统应在保存前验证数值范围
4. 当管理员提交成绩数据时，系统应将包含学号、课程编号、学期和分数的成绩记录保存到数据库
5. 当管理员完成成绩录入时，系统应显示成功确认信息及保存的记录数量

### 需求 5

**用户故事：** 作为管理员，我希望能够查询和统计成绩，以便分析学生的学习情况

#### 验收标准

1. 当管理员选择成绩查询功能时，系统应允许按课程、班级和学期进行筛选
2. 当管理员应用筛选条件并搜索时，系统应显示匹配的成绩记录及学生姓名和分数
3. 当管理员查看某门课程的成绩列表时，系统应计算并显示平均分、最高分和最低分
4. 当管理员请求及格率统计时，系统应计算分数在 60 分及以上的学生百分比
5. 当管理员查看成绩数据时，系统应在界面上显示所有统计信息

### 需求 6

**用户故事：** 作为管理员，我希望能够查询学生成绩，以便了解学生的学习成果

#### 验收标准

1. 当管理员选择学生成绩查询功能时，系统应允许输入学号进行查询
2. 当管理员输入学号并查询时，系统应显示该学生的所有成绩记录
3. 当管理员查看学生成绩列表时，系统应显示每条记录的课程名称、学期、分数和学分
4. 当管理员选择学期筛选时，系统应仅显示该学期的成绩
5. 当管理员查看学生成绩汇总时，系统应计算并显示该学生的总学分和平均分

### 需求 7

**用户故事：** 作为管理员，我希望能够管理教师信息，以便维护教师档案和课程分配

#### 验收标准

1. 当管理员选择添加教师功能时，系统应显示教师信息输入表单，包含工号、姓名、性别、所属系部和联系方式字段
2. 当管理员提交有效的教师信息时，系统应将教师记录保存到数据库并显示成功确认信息
3. 当管理员查看教师列表时，系统应显示所有教师及其分配的课程数量
4. 当管理员选择一位教师并选择编辑功能时，系统应允许修改除工号外的教师信息
5. 如果教师已分配课程，则系统应阻止删除该教师记录

### 需求 8

**用户故事：** 作为管理员，我希望系统有登录验证功能，以便保护数据安全

#### 验收标准

1. 当用户打开应用程序时，系统应显示包含用户名和密码字段的登录表单
2. 当管理员输入凭据并提交时，系统应根据数据库记录验证用户名和密码
3. 如果凭据无效，则系统应显示错误消息并阻止访问主要功能
4. 当管理员成功登录时，系统应显示主菜单界面
5. 当管理员选择退出功能时，系统应返回登录界面
