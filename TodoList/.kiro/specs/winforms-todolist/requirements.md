# 需求文档

## 简介

一个简单的 Windows Forms 待办事项应用，使用 C# 编写。所有代码都在一个窗体类中，使用 JSON 文件保存数据。适合大一学生学习，代码约1000行。

## 术语表

- **待办事项应用**: 管理日常任务的桌面程序
- **任务**: 一条待办事项
- **JSON**: 数据存储格式，易读易写

## 需求列表

### 需求 1：添加任务

**用户故事：** 作为用户，我想添加新任务，以便记录我需要做的事情。

#### 验收标准

1. 当用户输入任务内容并点击添加按钮时，应用应创建新任务并显示在列表中
2. 当用户尝试添加空任务时，应用应阻止添加
3. 当新任务添加成功后，应用应清空输入框

### 需求 2：标记完成

**用户故事：** 作为用户，我想标记任务为已完成，以便追踪我的进度。

#### 验收标准

1. 当用户点击任务的复选框时，应用应切换任务的完成状态
2. 当任务被标记为完成时，应用应以删除线样式显示该任务

### 需求 3：删除任务

**用户故事：** 作为用户，我想删除任务，以便移除不再需要的事项。

#### 验收标准

1. 当用户点击任务旁边的删除按钮时，应用应从列表中移除该任务
2. 当任务被删除后，应用应立即更新显示

### 需求 4：数据保存

**用户故事：** 作为用户，我想让任务自动保存，以便不会丢失数据。

#### 验收标准

1. 当任务被添加、完成或删除时，应用应将所有任务保存到 JSON 文件
2. 当应用启动时，应用应从 JSON 文件加载任务
3. 当 JSON 文件不存在时，应用应以空列表启动

### 需求 5：任务统计

**用户故事：** 作为用户，我想看到任务统计，以便了解我的进度。

#### 验收标准

1. 应用应显示任务总数
2. 应用应显示已完成任务数

### 需求 6：界面设计

**用户故事：** 作为用户，我想要一个简洁的界面，以便轻松使用应用。

#### 验收标准

1. 当应用启动时，应显示一个大小合适的窗口
2. 任务应以清晰、易读的格式显示
